modules.resource.Application module¶

Application module, defines the Application Class

Usage:

from modules.resources.Application import Application app = Application()

class modules.resource.Application.Application(data: Dict | None = None, *, id: int | None = None, num_procs: int = 1)¶

Bases: object

An application is defined as a graph of processus (array of arrays, potentially a networkx graph in the future).

It is initialized with empty data by default, usually populated with a list of processus, a matrix of processus links, and device deployment information.

Attributes:¶

idint

The ID of the application, generated automatically during initialization.

durationint

The duration of the application in chunks of 10 ms.

num_procsint

The number of processus in the application.

processus_listList[Processus]

A list of Processus objects representing the individual processus in the application.

proc_linksnp.ndarray

A matrix representing the bandwidth request over virtual links between processus.

deployment_infodict

A dictionary linking Processus objects to Device IDs.

class modules.resource.Device.Device(data: ~typing.Dict, routing_table_type=<class 'modules.routing.RoutingTable.RoutingTable'>)¶

Bases: object

Represents a computing device with limited resources and network capabilities.

A Device is characterized by resource limitations in terms of CPU, GPU, memory, and disk space. Each resource has an associated maximum limit and current usage level. The device also maintains a routing table to discover optimal paths to other devices in the network.

device_id¶

The unique identifier for this device.

Type:

int

location¶

The x, y, z coordinates of the device in the network.

Type:

Tuple[float, float, float]

resource_limit¶

A dictionary containing the resource limits for the device. Keys can be resource names like ‘CPU’, ‘GPU’, etc., and the values are the respective limits.

Type:

Dict[str, Union[int, float]]

current_resource_usage¶

A dictionary containing the current resource usage for the device.

Type:

Dict[str, Union[int, float]]

theoretical_resource_usage¶

A dictionary containing the theoretical resource usage for the device.

Type:

Dict[str, Union[int, float]]

resource_usage_history¶

A history of resource usage for each type of resource. Each history is a list of tuples, where the first element is the time, and the second is the resource usage at that time.

Type:

Dict[str, List[Tuple[int, Union[int, float]]]]

routing_table¶

A dictionary representing the routing table for this device. Each key is the device_id of a destination, and the value is a tuple (next_hop_id, distance).

Type:

Dict[int, Tuple[int, float]]

proc¶

A list containing processus that are running on this device.

Type:

Adds a new destination or update an existing one in the routing table Reminder - routing table element are : {destination:(next_hop, distance)}

Generates a rounting table progressively by adding devices Path are not considered, only next hop and distance Single path for now

Parameters:

  • distance_destination (int) – distance from device (self) to destination (destination_id), when passing through device (next_hop_id), distance is arbitrary, can be actual distance, number of hops, …

modules.resource.Processus module¶

class modules.resource.Processus.Processus(data: Dict[str, Any] | None = None, *, priority: int = 0)¶

Bases: object

The Processus class represents a sub-component of an Application and is responsible for managing its resource requirements and allocations.

This class provides methods for comparing Processus instances, combining their resource requirements, and generating JSON representations.

Comparison is based on an arbitrary resource priority order.
